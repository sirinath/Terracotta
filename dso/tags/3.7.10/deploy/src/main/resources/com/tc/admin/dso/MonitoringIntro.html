Terracotta Developer Console: Monitoring

Contains real-time, graphical system performance metrics and distributed garbage collection events to diagnose problems and recognize and interpret trends.

Runtime statistics
Real-time cluster statistics. Gauge the health of hosts, server instances, and clients. Monitor host resources, server-instance and client object faulting and flushing, and cache and transaction efficiency.
Garbage collection
Distributed garbage collection (DGC) removes objects that are no longer referenced on any client and have been made eligible for collection by Java GC. DGC runs on schedule, but can also be triggered manually. If the DCG's standard configuration may not be optimal for an application's characteristics, it can be tuned.
