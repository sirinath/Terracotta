Welcome to Terracotta Developer Console

The Terracotta Developer Console is a full-featured monitoring and diagnostics tool aimed at the development and testing phases of an application clustered with Terracotta. Use the Developer Console to isolate issues, discover tuning opportunities, observe application behavior with clustering, and learn how the cluster holds up under load and failure conditions.

The upper-left pane of the Developer Console contains a collapsible hierarchical navigation tree for accessing various features. Features are divided into these categories:

My Application
Provides feature-specific presentations relating to various aspects of your distributed application, including HTTP Sessions, Ehcache/Hibernate, and Quartz.
Monitoring
Contains real-time, graphical system performance metrics and distributed garbage collection events to diagnose problems and recognize and interpret trends.
Topology
Contains detailed information about the clients and server arrays. Mirror Groups (sets of active-"hot" standby server instances) are listed under Server Arrays, while Terracotta server instances are listed under their parent mirror group.
Platform
Contains detailed information about the underlying Distributed Shared Objects platform, used as the basis for the Express Ehcache, Quartz, and HTTP Session clustering product suites. Browse the Clustered Heap of shared objects and access Diagnostics that can be critical to understanding the operation of your cluster.
