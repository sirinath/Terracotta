Terracotta Developer Console: Clustered Heap

Contains tools for monitoring shared objects in the clustered heap. The clustered heap is the memory shared with Terracotta across JVMs. Shared object graphs are found in the clustered heap.

Object browser
Use the object browser to inspect objects currently in the clustered heap, for example to ensure that specified Terracotta roots are being clustered or that shared objects appear on clients as expected. The object browser displays shared object graphs for the entire cluster or by client, depending on how the filter is set.
Instance counts
Use the instance browser to determined which instrumented application classes have been instantiated on the clustered heap. The instantiated classes can be viewed in different formats, including tabular (with counts) and hierarchical.
